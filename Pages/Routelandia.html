Routelandia

Capstone project, a PHP backend supporting an Android app front end. Provides traffic statistics information for Portland OR, based on the Portal project.

The group project I was apart of for my computer science capstone. I was on a team of 7 members and we developed the Routelandia app and backend for the Portal Project. The app lets you select two points on a specific highway and will calculate the travel time in the past, present and predictions in the future. The sponsor only wanted an android app but some of the team members are continuing on with an iOS app, which will hopefully be released in the coming months.

The main idea behind the app was to help people decide when they should start to go home. Since traffic follows very cyclical patterns, the app will find these patterns and use them to predict how traffic is going to look in the future using data from the Oregon Department of Transportation.



Features
  • Dynamically Highlighting Highways
    • If a new highway gets added then it will automatically incorporated in the app
  • Travel time calculation for the past present and future
  • Recommended speed while traveling
  • Accuracy statistic
    • Detectors are taken offline for maintenance, this drastically affects our predictions
    • The accuracy number will let the user know how much to trust the predictions
My involvement/Responsibilities
  • Writing the Detectors API routes
  • Coordination between backend and frontend in developing API structure
  • Writing and implementing the stations pairing algorithm
    • Front end could only send us coordinates and not station IDs
    • I helped make a database query that would grab the nearest stations and it often grabbed at least 4 stations or more per point
    • I wrote an algorithm that figured out which station pair was the only "legal" one from the 8+ points
  • Lots of miscellaneous code writing and improvement
